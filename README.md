# Django Webtoon Project

## 📅 이번주 작업 요약

## 주요 작업 및 개선 내용

- **웹툰 데이터 동기화 및 플랫폼별 목록 구현**
    - 네이버, 카카오, 카카오페이지 웹툰 1~50페이지 자동 동기화 및 DB 저장
    - DB 비어있을 경우, 최초 접근 시 API를 통한 데이터 적재

- **웹툰 목록/상세 페이지 UI 고도화**
    - 반응형 그리드 및 카드형 레이아웃 적용
    - 이미지 로딩 중 스켈레톤(로딩 효과) 도입
    - 썸네일 로딩 실패 대응 추가

- **관심웹툰(즐겨찾기) 기능**
    - ManyToMany 필드로 간결하게 즐겨찾기 관계 구현
    - 웹툰 목록 및 마이페이지에서 별(⭐) 버튼으로 AJAX 즐겨찾기 토글
    - 관심웹툰(즐겨찾기) 해제 시 부드러운 카드 제거 및 UX 반영

- **마이페이지 핵심 구축**
    - 네이버 스타일 탭 구조: 관심웹툰/최근 본/내가 단 댓글(앞으로 확장 가능)
    - 관심웹툰 탭 내 플랫폼별 필터, 검색 기능, 그리드 출력 완성

- **회원가입/로그인 UI 및 기능 개선**
    - 반응형·모던 디자인의 로그인/회원가입 폼 적용
    - 회원가입 시 성별(남/여) 라디오 버튼으로 선택 필수
    - 가입 후 자동 로그인 및 바로 메인 페이지 이동 처리

---

## 이번주 어려웠던 점과 해결 사례

- **API 대량 데이터 반복/동기화**: break 조건, 중복 방지, 효율적 수집 고민
- **템플릿에서 authors 출력 오류**: DB와 뷰, 템플릿간 타입 혼선 해결
- **AJAX-즐겨찾기 토글**: 버튼 동작/카드 동적 제거, CSRF 및 UX 관리
- **회원가입폼 필드 확장**: 성별 선택 UI 자연스럽게 적용, 필수성 관리

## ✨ 앞으로의 개선/계획

- **실제 웹툰 사이트와 비슷한 부가 기능 추가**
  - 추천 웹툰, 남녀 인기순 등 원하는 방식으로 웹툰 정렬
  - 최근 본 웹툰, 내가 단 댓글 등 추가 및 탭 확장
  - 사용자 시청 이력 기반 추천 웹툰 리스트 최신화 등

- **Front-end 고도화**
  - 기본 Django 템플릿 → 추후 Vue 등 SPA 프론트엔드로 전환 예정
  - UX/UI 개선, 검색 고도화, 리스트/상세 페이지 분리

- **Back-end 구조화/고도화**
  - API 동기화, 에러/로깅 관리, 서비스 추상화
  - 웹툰 데이터 마이그레이션, 효율적 캐싱, 배치 동기화 개선

- **추천 시스템 구현**
  - 처음 회원가입하면 선호하는 웹툰 선택
  - 웹툰 기반으로 상단에 추천하는 웹툰 리스트업
  - 사용자의 웹툰 열람 기록을 바탕으로 추천 리스트업 업데이트 등

- **운영/배포**
  - GitHub 기반 코드관리 및 협업, Render 등 배포환경 자동화 준비
